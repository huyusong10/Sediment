# 金蝉脱壳实施指南

## 1. 概述

金蝉脱壳是谐振腔故障无损迁移的标准流程，用于在谐振腔即将坍缩或存在严重缺陷时，将其中的哈基米完整迁移至目标腔体，同时确保传输过程零损耗。

本指南适用于铁匠、调音师和守望者，在执行前需获得掌灯人或祭司团授权。

## 2. 前置条件

### 2.1 环境要求

| 条件 | 要求 | 验证方式 |
|------|------|----------|
| 目标谐振腔 | 已通过试音，嗡鸣度处于底噪水平 | 回音壁读数确认 |
| 热备份就绪 | 目标腔已预热至温存状态 | 千机匣状态检查 |
| 传输路径 | 走线完成，路径上无盲区 | 巡河验证通过 |
| 引雷针部署 | 潮涌能量引导设备就位 | 铁匠现场检查 |
| 守望者在岗 | 迁移全程需守望者监控静默态 | 值班表确认 |

### 2.2 授权要求

- **一级迁移**（单腔，哈基米量 < 1000 单位）：调音师可决策
- **二级迁移**（单腔，哈基米量 ≥ 1000 单位）：需掌灯人授权
- **三级迁移**（多腔，涉及织网变更）：需祭司团审批

## 3. 实施步骤

### 3.1 阶段一：准备（预计 15 分钟）

```
1. 确认源腔哈基米总量和晶格化状态
   └── 若未晶格化，先执行晶格化流程

2. 目标腔预检
   ├── 试音验证可用性
   ├── 嗡鸣度校准至底噪水平
   └── 饱和度确认为 0%

3. 部署引雷针
   ├── 安装于源腔与目标腔之间
   └── 测试引雷针引导能力

4. 启动热备份
   ├── 目标腔从温存切换至热备份
   └── 验证热备份接管延迟 < 200ms

5. 守望者就位
   └── 监控静默态和传输路径状态
```

### 3.2 阶段二：迁移（预计 5-30 分钟）

```
1. 启动风火轮加速跃迁
   └── 风火轮可将跃迁时间缩短 40%

2. 执行哈基米迁移
   ├── 源腔开始释放晶格化哈基米
   ├── 引雷针引导能量流向目标腔
   ├── 风火轮优化传输路径
   └── 目标腔接收并重新晶格化

3. 实时监控
   ├── 嗡鸣度变化曲线
   ├── 哈基米收支平衡（源腔减少量 = 目标腔增加量）
   └── 传输损耗率（目标 < 0.1%）

4. 跃迁完成验证
   ├── 源腔哈基米量降至 0
   ├── 目标腔哈基米量与源腔初始值偏差 < 0.1%
   └── 嗡鸣度稳定在安全弦范围内
```

### 3.3 阶段三：切换（预计 2 分钟）

```
1. 将目标腔从热备份切换为主腔
2. 更新织网拓扑（如需要）
3. 源腔标记为"待维修"
4. 守望者确认静默态已消除
5. 通知账房更新哈基米归属记录
```

## 4. 坍缩应急处理

如果在迁移过程中源腔发生坍缩（谐振腔失效导致哈基米释放）：

```
IF 检测到坍缩 THEN
    1. 引雷针全功率引导散逸能量
    2. 风火轮加速剩余哈基米跃迁至目标腔
    3. 守望者评估散斑污染范围
    4. 若散斑污染 > 阈值，启动清道夫流程
    5. 记录坍缩事件至账房
    6. 通知铁匠评估源腔可修复性
END IF
```

## 5. 验证清单

迁移完成后，逐项确认：

- [ ] 目标腔哈基米量 ≥ 源腔初始量 × 99.9%
- [ ] 目标腔嗡鸣度在共振峰区间内
- [ ] 目标腔镀层完好（溢彩状态）
- [ ] 传输损耗率 < 0.1%
- [ ] 织网拓扑已更新
- [ ] 账房记录完整
- [ ] 守望者确认系统状态正常
- [ ] 回音壁读数无盲区

## 6. 回退方案

若迁移失败，执行以下回退步骤：

```
1. 停止风火轮和引雷针
2. 将源腔中剩余哈基米重新晶格化
3. 将已迁移至目标腔的哈基米逆灌回源腔
4. 恢复原织网拓扑
5. 通知掌灯人和祭司团
6. 分析失败原因，修订迁移方案
```

## 7. 性能指标

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 迁移总耗时 | < 45 分钟 | 阶段一至阶段三总时长 |
| 哈基米损耗率 | < 0.1% | (源腔初始量 - 目标腔最终量) / 源腔初始量 |
| 风火轮加速比 | ≥ 1.4x | 使用风火轮 vs 不使用的时间比 |
| 跃迁成功率 | > 99.5% | 跃迁完成次数 / 总跃迁次数 |
| 热备份切换延迟 | < 200ms | 切换开始到目标腔就绪的时间 |
