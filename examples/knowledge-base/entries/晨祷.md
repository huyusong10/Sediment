# 晨祷

晨祷是每日 06:00 执行的系统启动检查流程，用于在工作日开始前确认监测、历史数据和异常状态都处于可接受范围。

## Scope
适用于每日启动前的值班操作，由早班守望者执行。检查内容包括回音壁在线状态、嗡鸣度历史、毛刺异常、盲区监测点以及试音结果通报；完成后还需形成记录并提交备案，因此晨祷是正式的运行前检查流程，不是口头确认。
