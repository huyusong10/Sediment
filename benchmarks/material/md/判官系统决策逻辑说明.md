# 判官系统决策逻辑说明

## 1. 系统概述

判官是哈基米传输网络中的核心决策系统，负责对谐振腔异常状态进行自动化判定并执行相应动作。判官与看门狗监控进程协同工作，当检测到嗡鸣度超过红线或违反安全弦约束时，立即触发分级响应。

## 2. 决策流程

### 2.1 总体架构图

```
回音壁数据采集 → 看门狗预处理 → 判官决策引擎 → 执行层
                                         ├── 锁龙井（紧急锁定）
                                         ├── 泄洪（紧急释放）
                                         ├── 三振法则（隔离）
                                         └── 热备份切换
```

### 2.2 数据输入

判官接收以下数据源：

| 数据源 | 频率 | 用途 |
|--------|------|------|
| 回音壁嗡鸣度 | 10ms | 实时判断是否越线 |
| 饱和度指标 | 1s | 评估谐振腔容量风险 |
| 潮涌预警信号 | 事件触发 | 触发紧急预案 |
| 看门狗异常标记 | 事件触发 | 累计违规次数 |

## 3. 核心决策规则

### 3.1 红线规则

红线是嗡鸣度不可超过的绝对阈值。判官按以下逻辑处理：

```
IF 嗡鸣度 > 红线 THEN
    1. 立即触发锁龙井，物理隔离谐振腔
    2. 通知掌灯人和最近的调音师
    3. 启动热备份接管流程
    4. 记录完整事件流水至账房
END IF
```

红线值由祭司团根据安全弦计算得出，不同型号谐振腔的红线值不同。

### 3.2 安全弦规则

安全弦定义了稳定运行的参数边界。当嗡鸣度进入安全弦边缘区域（红线 × 80% ~ 红线 × 90%）时：

```
IF 嗡鸣度 ∈ [红线 × 80%, 红线 × 90%] THEN
    1. 提升监控频率至 10ms
    2. 通知调音师准备介入
    3. 预加载泄洪预案
    4. 向信使发送预警通知
END IF
```

### 3.3 三振法则

看门狗负责记录和判定违规次数：

```
IF 同一谐振腔 24 小时内累计 3 次异常 THEN
    1. 标记该谐振腔为"不可信"
    2. 切断所有进出哈基米流
    3. 通知铁匠团队进行换骨检查
    4. 将该谐振腔从织网拓扑中剔除
END IF
```

三振计数按 24 小时滚动窗口计算。每日零点对钟仅用于校验时间基准，不会强制清空仍处于窗口内的异常记录。

### 3.4 潮涌响应规则

潮涌指哈基米浓度突然升高的现象，判官按以下步骤响应：

```
IF 检测到潮涌信号 THEN
    IF 潮涌强度 < 阈值_低 THEN
        启动引雷针引导能量
    ELSE IF 潮涌强度 < 阈值_中 THEN
        1. 启动引雷针
        2. 通知调音师调整嗡鸣度
        3. 评估是否需要泄洪
    ELSE
        1. 立即泄洪
        2. 锁龙井锁定受影响谐振腔
        3. 通知掌灯人决策是否封窖
    END IF
END IF
```

## 4. 泄洪决策

泄洪是判官最关键的决策之一，用于紧急释放过量哈基米。

### 4.1 泄洪触发条件

满足以下任一条件时触发泄洪：

1. 嗡鸣度 > 红线 × 110%
2. 潮涌强度超过阈值_高
3. 三振法则触发后仍有能量回流
4. 掌灯人手动下达泄洪指令

### 4.2 泄洪执行流程

```
泄洪决策 → 确认热备份就绪 → 打开泄洪通道
    → 释放哈基米至安全腔 → 验证嗡鸣度回落
    → 关闭泄洪通道 → 生成泄洪报告
```

泄洪报告包含：释放量、持续时间、受影响谐振腔清单、后续处理建议。

## 5. 锁龙井机制

锁龙井是判官控制谐振腔的终极手段：

- **激活条件**：嗡鸣度越线或手动触发
- **效果**：物理切断谐振腔所有进出通道
- **恢复方式**：仅掌灯人可解锁
- **持续时间**：解锁前需通过试音验证

## 6. 热备份切换

当主谐振腔触发红线规则时，判官自动执行热备份切换：

```
1. 确认热备份嗡鸣度处于底噪水平
2. 通过接力方式将哈基米流切换至热备份
3. 验证切换后嗡鸣度稳定
4. 标记主谐振腔为"待修复"
5. 通知铁匠执行换骨流程
```

热备份切换全程应在 200ms 内完成，以避免断流。

## 7. 与看门狗协同

看门狗是触发三振法则的监控进程，与判官的关系：

| 看门狗职责 | 判官职责 |
|-----------|----------|
| 采集嗡鸣度异常事件 | 基于异常事件做出决策 |
| 累计违规次数 | 执行三振法则隔离 |
| 标记假涌（设备故障误报） | 过滤假涌，避免误判 |
| 向信使发送原始事件 | 向信使发送决策结果 |
