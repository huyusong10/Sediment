# 当前设计：Web 界面

企业版 Sediment 的 Web 层分成两个相互独立但共享 API 的界面：

- 前台知识门户
- 后台管理台

两者不应混在一个导航层里，否则“浏览知识”和“治理知识”会互相污染。

## 1. 产品决策

当前设计选择自建 Web 前端，不把 Quartz 作为主产品壳。

理由：

- 前台和后台都需要动态 API、提交、审核、状态轮询和鉴权
- Quartz 更适合只读静态浏览，不适合作为审核流与管理工具的基础
- 一套自建前端更容易共享搜索、图谱、权限和条目数据模型

Quartz 在后续仍可作为可选静态导出目标，而不是主路径。

## 2. 前台知识门户

### 2.1 目标

前台面向普通用户，负责：

- 浏览知识库
- 搜索概念
- 查看条目全文
- 浏览图谱
- 提交材料和修订意见

### 2.2 页面结构

- `/`
  - 知识库概览
  - 热门概念
  - 最近更新
  - 提交入口
- `/search`
  - 全文搜索
  - 支持标题、别名、正文命中
  - 支持高亮和摘要片段
- `/entry/:name`
  - 条目全文
  - 来源元数据
  - Related 链接
  - 入链/出链概览
- `/graph`
  - 全局图谱
  - 局部邻域图
  - 节点点击跳转
- `/submit`
  - 纯文本概念/经验提交
  - 文档上传
  - 意见反馈

### 2.3 前台功能要求

- 支持全文搜索
- 支持图谱浏览
- 支持查看 Markdown 正文渲染结果
- 提交表单要求填写姓名
- 后端自动记录 IP
- 至少按 IP 做每分钟 1 次限流

## 3. 后台管理台

### 3.1 目标

后台面向 `committer` 和平台维护者，负责：

- 审核提交
- 在线编辑条目
- 常驻展示 health
- 发起和查看 ingest / tidy 任务
- 审阅 diff 和 patch

### 3.2 页面结构

- `/admin`
  - 未登录时显示登录页
  - 总览卡片
  - health 摘要
  - 待审核提交数
  - 运行中任务数
  - 系统状态与限流配置
- `/admin/submissions`
  - 待 triage 提交
  - 已 triage 提交
  - 已接受 / 已拒绝提交
- `/admin/submissions/:id`
  - 原始材料
  - 提取文本
  - 提交者与时间
  - 审核动作
- `/admin/health`
  - 问题总览
  - 按类型和严重度筛选
  - 跳转到具体条目
- `/admin/jobs`
  - ingest / tidy 任务列表
  - 运行日志
  - 失败重试
  - 取消中任务
  - 陈旧任务恢复可见性
- `/admin/reviews/:id`
  - diff 查看
  - 理由与上下文
  - 批准 / 拒绝 / 请求修改
- `/admin/entries/:name/edit`
  - 在线 Markdown 编辑
  - 预览
  - 校验结果

### 3.3 后台必须具备的能力

- 在线编辑正式条目
- 查看 health 结果并常驻刷新
- 查看结构异常条目、低置信条目、缺失段落条目
- 查看未进入正式知识库的提交
- 一键对问题条目创建 tidy 任务
- 审阅 Agent 生成的 patch
- 显示审计日志
- 显示系统状态、队列模式和限流配置
- 支持任务 cancel / retry

## 4. 共享组件

前台和后台至少共享以下组件：

- 条目摘要卡片
- Markdown 渲染器
- 图谱节点侧边栏
- 搜索结果列表
- 来源和状态标签

## 5. 图谱设计

图谱不应只是一个装饰性页面，而应同时服务于：

- 前台导航
- 后台诊断孤岛与弱连接条目

图谱节点至少区分：

- `concept`
- `lesson`
- `placeholder`
- `index`

图谱边至少区分：

- `Related`
- `index_link`

## 6. 搜索设计

搜索排序建议优先考虑：

1. 标题命中
2. 别名命中
3. 摘要命中
4. 正文命中
5. canonical / 健康加权

前台默认隐藏未合并草案和缓冲区内容。

后台可选择切换查看：

- 仅正式知识层
- 正式知识层 + 待审核草案

## 7. 编辑器设计

在线编辑器应支持：

- Markdown 编辑
- 即时预览
- frontmatter 保持
- 调用后端校验
- 合并前 diff 预览

后台在线编辑是受控直接编辑路径，不替代 submit / review 主流程。

## 8. 权限设计

前台权限：

- 普通用户可浏览、搜索、查看和提交

后台权限：

- `committer` 可 triage、发起任务、审核、在线编辑、合并
- 平台维护者可管理规则、限流、任务重试和系统设置
- Admin Web 登录建议使用同站 session cookie，脚本侧保留 Bearer token

## 9. 前端技术建议

推荐：

- `React + TypeScript`
- 门户与后台共享同一组件库
- 图谱使用 `Cytoscape.js`
- 编辑器使用 `Monaco`

重点不是追求大而全，而是尽早把“搜索、提交、审核、diff、health”这五条链路跑通。
