## 本轮优化内容

### 1. source_file_map 扩展（server.py）

**问题**: `role_permissions.yaml` 映射到 `['掌灯人', '权限体系']`，但问题常问具体角色（外乡人、祭司团等）。

**修改**:
- `role_permissions.yaml` 扩展为包含所有角色：`['外乡人', '掌灯人', '祭司团', '渡鸦', '铁匠', '园丁', '听风者', '守望者', '老把式', '角色权限配置', '权限体系']`
- `role_permissions` 同样扩展
- `角色权限配置` 同样扩展
- 其他文件映射也增加了更多候选条目（alert_rules → 潮涌, metric_definitions → 纯度/账房 等）
- `tracer.py` 增加 `溯光追踪代码`
- `echo_wall.py` 增加 `照妖镜`
- `cleaner.py` 增加 `散斑`

### 2. 语义规则扩展（server.py）

新增语义规则匹配低分问题：
- `清浊比.*下降.*毛刺|镀层.*老化|照骨灯` → 散斑（Q95）
- `镀层.*老化|换羽` → 换羽（Q95）
- `隐身衣|不能完全.*避免|账房.*审计` → 账房（Q96）
- `Q3.*事故|判官.*延迟|假涌.*误判` → 假涌（Q52, Q97）
- `幽灵读数.*诊断|照妖镜` → 照妖镜（Q51）
- `培训.*认证|外乡人.*入职` → 外乡人（Q99）

### 3. 引用条目优先级提升（server.py）

**问题**: 即使引用条目（如外乡人）被选中，排序后可能被其他高分条目（如掌灯人）排在后面。

**修改**: 在 `selected_entries.sort()` 之后，如果存在直接引用的条目，将其移到列表最前面，确保其内容优先用于答案合成。

### 4. 宽泛问题检测扩展（server.py）

新增 broad_indicators：
- `从.*看` — 捕获"从XX看"类推断问题
- `结合.*和` — 捕获多文档综合问题
- `为什么.*中` — 捕获"为什么Q3事故中..."类问题
- `事故.*为什么|为什么.*事故|根因|复盘` — 捕获事故分析问题
- `培训.*认证|从入职到|经过哪些` — 捕获培训流程问题
- `优势|相比.*区别|手动.*对比` — 捕获对比问题
- `综合.*文档|多份文档|综合推断` — 捕获综合推断问题

### 5. 细节问题(is_detail_question) 合成优化（server.py）

**问题**: `is_detail_question` 被定义但从未在答案合成中使用。

**修改**: 对细节问题（多少种/哪些/类型），使用条目的完整内容而非仅定义段+上下文，确保枚举信息完整。同时扩展检测正则添加 `类型|种类|消息`。

### 6. 引用条目答案合成优化（server.py）

对引用条目的内容提取进行优化：
- 始终包含定义段 + 上下文段
- 对权限类问题，额外提取权限/职责/操作/限制段落
- 对提到特定子概念的问题，额外提取相关内容行

### 7. TODO 问题条目收集增强（server.py）

对 TODO 问题，除了扫描所有条目外，还主动检查常见代码相关条目（谐振腔、调音师、看门狗、渡鸦、千机匣、账房等）是否包含 TODO/未实现/待实现/实际环境 信息，全部加入主词条。

### 8. 跨文档关联和因果链编码（isolated_build.py）

新增 ingest prompt 的"跨文档关联和因果链"章节，要求摄入 Agent 在条目中显式编码：
- **症状-原因链**: X可能是由Y导致的 / Y会导致X症状
- **工具-问题链**: 可使用A工具进行诊断
- **限制-影响链**: A的局限性：B
- **事故-根因链**: C的根因：D / D曾导致C事故
- **权限-操作链**: 逐字引用权限明细（能做什么、不能做什么）
- **配置-行为链**: G受F配置影响
- **诊断方法**: 故障类条目必须包含"如何检测"

提供了具体示例：
- 照妖镜 → 用于检测幽灵读数
- 幽灵读数 → 可使用照妖镜诊断
- 镀层晦暗 → 导致清浊比下降和散斑
- 隐身衣 → 可避免回音壁检测，但无法避免账房审计
- 账房 → 可检测到隐身衣无法掩盖的收支不平衡

### 9. 角色类条目特殊要求（isolated_build.py）

新增角色类条目的特殊要求：
- 定义段: XX是[职责]的[角色类型]，拥有/仅拥有[权限范围]
- 上下文中必须包含：权限等级、能做什么、不能做什么、限制条件、晋升路径
- 外乡人示例：只能查看嗡鸣度数据，不能执行启明/泄洪/织网等操作，不能修改配置，试用期90天
