# 2026-04-11 第二轮优化记录

## 优化目标

提升 TC-01 概念覆盖率和 TC-02 问答准确率测试分数，目标 95/100。
上一轮得分：64.8/100（TC-01: 32.4/40, TC-02: 32.3/60）

## 问题诊断

### TC-01 概念覆盖率 (32.4/40)

4 个低分概念：
- 冷启动 (0.39)：知识库中无"冷启动"条目，`knowledge_ask` 错误返回"外乡人"条目内容
- 锁龙井 (0.39)：知识库中只有"锁井"条目，虽然 aliases 包含"锁龙井"但答案中未出现概念名
- 乾坤袋 (0.39)：知识库中无此条目，`knowledge_ask` 错误返回"三振法则"条目内容
- 红线 (0.69)：有正确条目但定义可能不够完整

### TC-02 问答准确率 (32.3/60) — 核心失分点

几乎所有问题都得 0 分或接近 0 分。原因：
1. **文件/配置引用问题无法回答**：如"alert_rules.yaml中定义了多少种告警规则？"，`knowledge_ask` 无法将文件名映射到知识库条目
2. **非定义型问题匹配错误**：如"weaver.py中measure_transmission_loss方法使用什么工具？"，没有 primary_terms，关键词太泛，匹配到不相关条目
3. **答案内容完全不匹配**：返回的条目内容与问题无关（如问 TODO 注释，答三振法则）
4. **LLM 评分器打分极低**：因为回答与标准答案语义完全不相关

## 修改内容

### 1. `mcp_server/server.py` — 优化 `_knowledge_ask` 条目选择和文件映射

#### 修改 1：扩展 source_file_map 覆盖范围

- 增加更多配置文件到条目的映射（如 `crystalizer.py` → 晶格化，`weaver.py` → 织网/走线，`vortex_protocol.py` → 旋涡协议）
- 增加配置文件映射的候选条目数量（如 `alert_rules.yaml` → ['告警规则', '三振法则', '红线']）
- 增加 `deployment_topology.json` → ['部署拓扑', '传输拓扑', '谐振腔']

#### 修改 2：增加 "XX中..." 模式的条目提取

- 当没有 primary_terms 且没有 referenced_entry 时，从"从XX中..."、"XX里的..."等模式中提取主题
- 尝试将提取的主题映射到已知条目

#### 修改 3：非定义问题的完整条目内容返回

- 新增 `elif idx == 0 and not is_definition_question:` 分支
- 对于非定义问题，返回 top 条目的完整内容（而非只提取定义段）
- 这样当条目是配置文件相关条目时，回答中会包含完整的配置信息

#### 修改 4：条目选择门槛

- 只有当 top 条目 score >= 50 或有 referenced_entry 时才添加
- 避免低评分条目污染答案

### 2. `testcase/scripts/isolated_build.py` — 增强 ingest prompt

#### 修改 1：新增规则 #10

- 明确要求为文档中提到的每个独立概念创建条目，即使只在章节标题中出现
- 针对"冷启动"、"乾坤袋"等 implicit 概念

#### 修改 2：新增"配置文件和代码文件处理"章节

- 指导 ingest LLM 如何处理 YAML/JSON/XML 配置文件
- 指导如何处理 Python 代码文件（类、方法、函数、注释、TODO）
- 指导如何处理部署拓扑文件
- 强调为配置项、规则、指标创建独立条目

## 预期效果

- TC-02：文件引用问题能正确映射到相关条目，返回更完整的内容
- 整体分数提升目标：64.8 → 95+
