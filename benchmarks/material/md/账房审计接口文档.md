# 账房审计接口文档

## 1. 概述

账房是记录哈基米收支的审计系统，所有哈基米的产生、传输、消耗和归档均通过账房留痕。本文档描述账房对外提供的 RESTful API 接口。

### 1.1 协议说明

所有接口基于 **旋涡协议** 进行哈基米传输验证。旋涡协议是哈基米传输的标准协议，确保数据在传输过程中的完整性和可追溯性。

### 1.2 认证方式

所有请求需在 Header 中携带溯光令牌：

```
Authorization: Bearer <溯光令牌>
```

溯光技术用于追踪哈基米来源，令牌中包含请求者的身份标识和权限范围。

## 2. 基础信息

- Base URL: `https://ledger.hakimi.internal/api/v2`
- 数据格式: JSON
- 字符编码: UTF-8
- 对钟要求: 请求时间戳与服务器时间偏差不得超过 30 秒

## 3. 接口列表

### 3.1 查询哈基米流水

查询指定时间范围内的哈基米收支记录。

**请求**

```
GET /transactions
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| start_time | string | 是 | 起始时间 ISO 8601 |
| end_time | string | 是 | 结束时间 ISO 8601 |
| cavity_id | string | 否 | 谐振腔 ID 过滤 |
| min_purity | float | 否 | 最低清浊比过滤 |

**响应**

```json
{
  "code": 0,
  "data": {
    "transactions": [
      {
        "tx_id": "tx_20260401_0001",
        "timestamp": "2026-04-01T08:00:00Z",
        "cavity_id": "cav_001",
        "amount": 1250.5,
        "direction": "in",
        "protocol": "旋涡协议",
        "purity_ratio": 19.0,
        "source_trace": "溯光追踪路径..."
      }
    ],
    "total": 1,
    "has_more": false
  }
}
```

### 3.2 查询清浊比统计

获取指定谐振腔的清浊比（纯净哈基米与散斑的比例）统计。

**请求**

```
GET /purity-stats/{cavity_id}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| cavity_id | string | 是 | 谐振腔 ID |
| period | string | 否 | 统计周期：hour/day/week，默认 day |

**响应**

```json
{
  "code": 0,
  "data": {
    "cavity_id": "cav_001",
    "period": "day",
    "avg_purity": 0.95,
    "min_purity": 0.89,
    "max_purity": 0.99,
    "scatter_count": 12,
    "last_updated": "2026-04-01T12:00:00Z"
  }
}
```

### 3.3 审计暗流事件

查询检测到的暗流（不经过旋涡协议的非法传输）事件。

**请求**

```
GET /darkflow-events
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| start_time | string | 否 | 起始时间 |
| severity | string | 否 | 严重级别：low/medium/high |
| status | string | 否 | 处理状态：pending/resolved |

**响应**

```json
{
  "code": 0,
  "data": {
    "events": [
      {
        "event_id": "df_20260401_003",
        "timestamp": "2026-04-01T03:15:00Z",
        "source_cavity": "cav_007",
        "target_cavity": "cav_012",
        "estimated_amount": 45.0,
        "detection_method": "溯光差异分析",
        "severity": "high",
        "status": "pending",
        "assigned_to": "渡鸦"
      }
    ]
  }
}
```

### 3.4 晚课归档查询

查询每日晚课（关闭前数据归档）的执行记录和结果。

**请求**

```
GET /archivals
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| date | string | 否 | 归档日期 YYYY-MM-DD |
| status | string | 否 | 执行状态：success/failed/partial |

**响应**

```json
{
  "code": 0,
  "data": {
    "archivals": [
      {
        "archival_id": "arch_20260401",
        "date": "2026-04-01",
        "start_time": "2026-04-01T22:00:00Z",
        "end_time": "2026-04-01T22:15:30Z",
        "status": "success",
        "records_count": 15832,
        "data_size_mb": 245.6,
        "checksum": "sha256:abc123..."
      }
    ]
  }
}
```

### 3.5 时间同步校验

用于对钟（校准时间同步）的接口，确保各节点时间一致。

**请求**

```
POST /time-sync
```

**请求体**

```json
{
  "client_timestamp": "2026-04-01T08:00:00.123Z",
  "node_id": "node_005"
}
```

**响应**

```json
{
  "code": 0,
  "data": {
    "server_timestamp": "2026-04-01T08:00:00.456Z",
    "drift_ms": 333,
    "is_synced": true,
    "max_allowed_drift_ms": 30000
  }
}
```

### 3.6 无字碑调试模式

无字碑是不记录日志的调试模式，仅在调试环境中可用。

**请求**

```
POST /debug/no-log
```

**请求体**

```json
{
  "operation": "test_transfer",
  "duration_seconds": 60,
  "authorized_by": "掌灯人"
}
```

**响应**

```json
{
  "code": 0,
  "data": {
    "session_id": "debug_001",
    "expires_at": "2026-04-01T09:00:00Z",
    "log_suppressed": true
  }
}
```

> **警告**：无字碑模式下的操作不会被账房记录，仅限授权调试使用。

## 4. 信使通知接口

账房通过信使向订阅方推送审计事件通知。

### 4.1 订阅通知

```
POST /notifications/subscribe
```

**请求体**

```json
{
  "subscriber_id": "watcher_001",
  "event_types": ["darkflow_detected", "purity_drop", "archival_completed"],
  "callback_url": "https://watcher.hakimi.internal/webhook"
}
```

### 4.2 通知格式

信使推送的通知格式如下：

```json
{
  "event_type": "darkflow_detected",
  "timestamp": "2026-04-01T03:15:00Z",
  "payload": {
    "event_id": "df_20260401_003",
    "severity": "high",
    "details": "检测到未经旋涡协议的哈基米传输"
  },
  "trace_id": "溯光追踪ID"
}
```

## 5. 错误码

| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 4001 | 溯光令牌无效或过期 |
| 4002 | 对钟失败，时间偏差过大 |
| 4003 | 旋涡协议验证失败 |
| 4004 | 权限不足 |
| 5001 | 账房内部错误 |
| 5002 | 数据归档中，暂不可查询 |
