# 场景：先看健康队列，再在控制台中修复文档

**前置**：已登录管理台；知识库里至少有一个治理问题文档

**步骤**：
1. 打开 `/admin/files`
2. 先停留在左侧入口区，切换到“健康队列”Tab
3. 观察按文档聚合后的健康卡片，选择一个存在治理问题的文档
4. 确认右侧编辑控制台自动载入该文档，并把顶部信息区切到“关联问题”
5. 在顶部固定信息区里切换查看“元数据”和“关联问题”，确认内容超出时只在区块内部滚动，然后返回 Markdown 编辑区修改内容但先不要保存
6. 点击“恢复”，确认编辑内容一键回到最近一次载入状态
7. 再次修改内容但先不要保存，点击“重新载入”
8. 在未保存确认弹层中先取消，确认当前编辑内容仍保留
9. 再次点击“重新载入”并确认放弃修改，确认内容回到服务端最新版本
10. 再次修改内容但先不要保存，尝试从左侧再次打开另一个文档
11. 在未保存确认弹层中先取消，确认当前编辑内容仍保留
12. 再次切换文档并确认放弃修改
13. 点击“预览”按钮，确认弹窗中的 Markdown 渲染反映当前编辑内容，再关闭弹窗

**预期**：
- 左侧入口区不会把索引导航、健康队列和编辑器堆成同一纵向长页面
- 从健康队列进入后，右侧编辑控制台无需滚到页面底部即可保存
- “元数据”和“关联问题”位于编辑器上方，通过固定尺寸的 Tab 区切换，不会因为内容多少改变控制台整体布局
- 预览以弹窗出现，并展示当前未保存的实时正文内容，不再把 frontmatter 原样显示出来
- “恢复”会把当前文档回退到最近一次载入/保存后的内容，不需要刷新页面，也不会触发服务端写入
- “重新载入”会重新从服务端获取当前文件；若本地有未保存修改，必须先确认放弃后才能继续
- 未保存内容不会因为切换文档而静默丢失
